# 浏览器级测试报告

## 1. 测试目标

本项目将浏览器级验证作为功能完成的重要标准。测试目标不是单独验证接口返回，而是确认真实用户在页面上的操作链路可以正常完成。

## 2. 测试环境

- 浏览器引擎：Playwright Chromium
- Python 环境：`qdx_bishe`
- 前端地址：`http://localhost:5173`
- 后端地址：`http://localhost:8080`
- 数据库：MySQL 8

## 3. 执行命令

```bash
conda run -n qdx_bishe python tests/browser/full_browser_smoke.py
```

## 4. 覆盖范围

### 4.1 教师端

- 登录
- 教师首页加载
- 知识点管理页加载
- 题库管理页加载
- 手动新增单选题
- AI 题目草稿生成
- AI 模型切换
- AI 组卷草稿生成
- 保存试卷草稿
- 从试卷列表发布草稿试卷
- 查看试卷成绩
- 查看学生答卷详情
- 查看班级统计

### 4.2 学生端

- 登录
- 查看“我的考试”
- 进入考试详情页
- 完成单选题与简答题作答
- 提交试卷
- 查看学习分析

### 4.3 AI 相关链路

- AI 单选题草稿生成
- AI 组卷草稿生成
- AI 主观题评阅

## 5. 本次验证结果

### 5.1 自动化结果

- 后端测试：`44` 个测试通过
- 前端构建：通过
- 浏览器级全链路测试：通过

### 5.2 已验证的关键闭环

1. 教师可使用 AI 生成组卷草稿。
2. 教师可将试卷先保存为草稿，再从列表发布。
3. 学生能在考试列表中看到已发布试卷。
4. 学生提交后系统能返回客观题分、主观题分和总分。
5. 教师能查看该试卷的成绩明细和学生答卷详情。
6. 班级统计页能看到该试卷带来的统计数据。

## 6. 数据清理说明

浏览器级测试会临时创建题目、试卷和作答记录。测试完成后已执行清理脚本，确保数据库中不保留 `浏览器测试...` 命名的数据。

## 7. 结论

当前系统的核心业务流程已经完成浏览器级验证，具备毕业设计演示条件。测试方式覆盖真实页面交互，比单纯接口测试更接近最终使用场景。
